# Rules for working with LLMs on the JS client (`client/js/`)

## Pre-push checks

The repo does **not** have a pre-commit hook for JS/TS code in `client/js/`.
CI enforces `prettier`, `eslint`, TypeScript compilation, and the TypeDoc
build (`--treatWarningsAsErrors`). Run these locally before pushing:

```bash
cd client/js
npx tsc --noEmit                       # type check
npm run format:check                   # prettier
npm run lint                           # eslint
npm run docs                           # typedoc (warnings become errors)
npx vitest run --no-coverage           # unit + mock tests
```

If `format:check` reports issues, fix in place with:

```bash
npx prettier --write src test examples
```

Common pitfalls that CI will catch:

- **Prettier formatting** — line wrapping in multi-line function calls,
  trailing commas, etc. `prettier --write` is the canonical fixer.
- **`@typescript-eslint/no-deprecated`** — calling an `@deprecated` symbol
  triggers an error. For SDK-internal call sites that need a deprecated API,
  prefer rewriting to use the underlying internal helper. For test code that
  intentionally exercises the deprecated surface, suppress with
  `// eslint-disable-next-line @typescript-eslint/no-deprecated -- <reason>`
  immediately above the offending line, or `/* eslint-disable
@typescript-eslint/no-deprecated */` at the top of a file that is
  predominantly testing deprecated APIs. `test/legacy/**` is already
  exempted in `eslint.config.js`.
- **TypeDoc warnings = CI failure.** If a non-public field or type is
  referenced by a public symbol, TypeDoc warns that the referent isn't
  documented. Fix by either exposing the referent or hiding the referrer:
  - Prefix internal class fields with an underscore (`_mountOptions`) — by
    convention TypeDoc treats `_`-prefixed members as private.
  - Add `@hidden` to the JSDoc block to fully exclude from docs (in addition
    to `@internal` for tooling that respects it).

## Deprecating a public symbol

1. Add an `@deprecated <replacement>` line to the JSDoc block on the symbol.
2. Update every internal caller (in `client/js/src/`) so it stops calling the
   deprecated symbol — typically by reading an underlying internal field or
   helper directly, since ESLint will otherwise flag the use.
3. Update tests with the `eslint-disable` directives described above when
   they need to exercise the deprecated symbol.
4. Add a note to `client/CHANGELOG_GO_JS.md`.

## Internal-only helpers

If a function needs to be reachable from multiple modules under
`client/js/src/` but is not part of the public surface, export it from its
defining module and add `@internal` (and `@hidden` if it should also be
excluded from docs) to the JSDoc.

## Protobuf

`client/js/proto/modal_proto/` is generated by `client/js/scripts/gen-proto.sh`
and **is** checked in. Regenerate when the underlying `.proto` schema
changes, and commit the resulting `api.ts`.
